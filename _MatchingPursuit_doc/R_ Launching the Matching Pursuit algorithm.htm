empi.execute {MatchingPursuit}R Documentation

Launching the Matching Pursuit algorithm

Description

Runs the empi program for the given data.

Usage

empi.execute(signal, sampling.rate, empi.options = NULL, write.to.file = FALSE)

Arguments

signal

Must be a data frame: rows = samples for all channels, columns = channels. The data frame must have column names (channel names).

sampling.rate

Sampling rate of the given signal (must be the same for every channel).

empi.options

If NULL, the empi program runs with "-o local --gabor -i 50". Otherwise, user can spacify any command-line options. See README.md file after downloading the empi program using empi.download function.

write.to.file

If TRUE, the SQLite database file empi.db will be created and saved in the current directory. This file stores the results of signal decomposition using the Matching Pursuit algorithm.

Details

The empi program (sources and binaries for different OSs) is available for download at https://github.com/develancer/empi. Details are presented in a journal paper: Różański, P.T. (2024). empi: GPU-Accelerated Matching Pursuit with Continuous Dictionaries. ACM Transactions on Mathematical Software, vol.50, issue = 3, pp. 1-17, https://doi.org/10.1145/3674832.

Value

Results of signal decomposition using the Matching Pursuit algorithm. If write.to.file=TRUE additionally, these results are written to the empi.db disk file in the working directory.

Examples

Run examples

## Not run:
file <- system.file("extdata", "sample1.csv", package = "MatchingPursuit")
signal <- read.csv(file, header = TRUE)

if (interactive()) {
empi.out <- empi.execute (
  signal = signal,
  sampling.rate = 128,
  empi.options = NULL,
  write.to.file = FALSE
)
}
## End(Not run)
